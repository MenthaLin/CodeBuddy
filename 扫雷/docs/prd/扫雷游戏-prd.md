# 扫雷游戏（Minesweeper）- 产品需求文档

## 1. 产品概述

### 1.1 产品简介
开发一个经典的扫雷游戏网页版，玩家需要在包含隐藏地雷的网格中，通过逻辑推理找出所有地雷的位置并标记它们，同时避免点击到地雷。支持用户登录（昵称），记录用户的游戏历史、胜负统计和最佳成绩，并提供排行榜功能。

### 1.2 目标用户
- 休闲游戏玩家
- 喜欢逻辑推理的玩家
- 所有年龄段用户

### 1.3 核心玩法
- 游戏在一个矩形网格上进行，网格中随机分布着若干地雷
- 玩家点击格子进行探索，如果点到地雷则游戏失败
- 如果点到安全格，显示周围8格中的地雷数量
- 玩家可以通过右键标记可疑的地雷位置
- 当所有非地雷格都被打开时，游戏胜利

## 2. 功能需求

### 2.1 P0 核心功能（必须实现）

#### F1 - 游戏棋盘渲染
- 渲染一个可交互的网格棋盘
- 支持三种难度级别：
  - 初级：9×9 网格，10 个地雷
  - 中级：16×16 网格，40 个地雷
  - 高级：30×16 网格，99 个地雷
- 格子有三种状态：未打开、已打开、已标记旗帜
- 已打开的格子显示周围地雷数量（数字 1-8 用不同颜色标识）

#### F2 - 左键点击揭晓
- 左键点击未打开的格子进行揭晓
- 如果点到地雷：游戏结束，显示所有地雷位置
- 如果点到的格子周围无地雷（数字为0）：自动展开相邻安全区域（Flood Fill）
- 如果点到的格子周围有地雷：仅显示该格子的数字

#### F3 - 右键标记/取消旗帜
- 右键点击未打开的格子：放置旗帜标记（🚩）
- 再次右键已标记的格子：取消标记
- 已打开的格子不能标记
- 显示剩余地雷计数器（总地雷数 - 已标记旗帜数）

#### F4 - 游戏胜负判定
- **胜利条件**：所有非地雷格均被打开
- **失败条件**：点击到地雷格
- 游戏结束后禁用所有操作
- 胜利时显示祝贺信息，记录用时
- 失败时显示所有地雷位置，标出玩家点击的那颗雷

#### F5 - 计时器
- 从第一次点击开始计时
- 游戏进行中实时显示已用时间（格式：MM:SS）
- 游戏结束后停止计时

#### F6 - 新游戏/重新开始
- 提供"新游戏"按钮，可随时重新开始
- 可选择不同难度重新开始
- 第一次点击保证不会是地雷（安全首点）

---

### 2.2 P1 次要功能

#### F7 - 双击/和弦快速展开
- 在已打开的数字格上同时按左右键（或双击），如果周围旗帜数等于该数字，自动打开周围未标记格
- 这是经典扫雷的进阶操作，提升熟练玩家的游戏体验

#### F8 - 用户系统（登录/注册）
- 用户通过输入昵称进行登录/注册（简单方式，无需密码）
- 登录后显示当前用户昵称
- 支持切换用户（退出登录）
- 登录状态持久化（localStorage 缓存，Supabase 存储）

#### F9 - 游戏记录存储
- 每局游戏结束后，自动上传游戏记录到 Supabase 数据库
- 记录内容：用户 ID、难度、游戏结果（胜利/失败）、用时（秒）、完成日期
- 用户可在"我的记录"页面查看个人游戏历史
- 显示个人统计：总场次、胜率、各难度最佳时间

#### F10 - 排行榜
- 提供全局排行榜页面
- 按难度分别排行，以最佳时间升序排列（仅胜利记录上榜）
- 显示排名、昵称、难度、最佳时间、完成日期
- 支持分页查看

#### F11 - 最高分/最佳时间记录
- 完成游戏时若刷新个人记录，显示"新纪录！"提示
- 数据存储在 Supabase（优先）和 localStorage（离线缓存）

#### F12 - 游戏暂停
- 提供暂停按钮，暂停时隐藏棋盘内容
- 再次点击恢复游戏

#### F13 - 音效反馈
- 揭晓格子：点击音效
- 标记旗帜：插旗音效
- 踩雷：爆炸音效
- 胜利：胜利音效

## 3. 非功能需求

### 3.1 性能要求
- 游戏初始化（含地雷随机布局）应在 100ms 内完成
- 大范围自动展开（Flood Fill）应在 50ms 内完成
- 界面响应时间 < 16ms（60fps）
- 数据库查询响应 < 500ms

### 3.2 兼容性
- 支持现代浏览器：Chrome 90+, Firefox 88+, Edge 90+, Safari 14+
- 支持桌面端和移动端触屏操作

### 3.3 可用性
- 界面简洁直观，符合经典扫雷视觉风格
- 所有操作提供即时视觉反馈
- 响应式设计，适配不同屏幕尺寸

### 3.4 安全性
- 使用 Supabase RLS（Row Level Security）保护用户数据
- 用户只能读取自己的游戏记录
- 排行榜为公开只读

## 4. 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端框架 | **原生 HTML/CSS/JavaScript** | 轻量级，无框架依赖，适合单一游戏应用 |
| 构建工具 | 无需构建工具 | 直接使用 ES Module |
| 样式方案 | **原生 CSS + CSS Grid** | 棋盘布局天然适合 CSS Grid |
| 状态管理 | 游戏状态对象（Vanilla JS） | 单一状态对象管理游戏状态 |
| 后端服务 | **Node.js + Express** | 提供 API 接口，对接 Supabase |
| 数据库 | **Supabase (PostgreSQL)** | 存储用户数据、游戏记录、排行榜 |
| 用户认证 | Supabase Auth（匿名登录 + 昵称） | 简化的用户系统 |
| 音效 | **Web Audio API** 或简单音频文件 | P1 功能 |

## 6. 数据模型

### 6.1 游戏状态（GameState）
```javascript
{
  board: Cell[][],       // 二维数组，棋盘格子
  rows: number,          // 行数
  cols: number,          // 列数
  mines: number,         // 地雷总数
  flagsPlaced: number,   // 已放置旗帜数
  state: 'idle' | 'playing' | 'won' | 'lost',
  startTime: number|null,// 开始时间戳
  elapsedTime: number,   // 已用时间（秒）
  firstClickDone: false  // 是否已完成首次点击
}
```

### 6.2 格子（Cell）
```javascript
{
  isMine: boolean,       // 是否是地雷
  isRevealed: boolean,   // 是否已揭晓
  isFlagged: boolean,    // 是否已标记旗帜
  adjacentMines: number  // 周围地雷数（0-8）
}
```

### 6.3 用户（Supabase 表: users）
```sql
CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  nickname VARCHAR(50) NOT NULL UNIQUE,
  created_at TIMESTAMPTZ DEFAULT NOW()
);
```

### 6.4 游戏记录（Supabase 表: game_records）
```sql
CREATE TABLE game_records (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  user_id UUID REFERENCES users(id) NOT NULL,
  difficulty VARCHAR(10) NOT NULL CHECK (difficulty IN ('easy', 'medium', 'hard')),
  result VARCHAR(10) NOT NULL CHECK (result IN ('win', 'loss')),
  time_seconds INT NOT NULL,
  completed_at TIMESTAMPTZ DEFAULT NOW()
);
```

## 8. 界面设计要点

### 8.1 布局
```
┌──────────────────────────────────────────┐
│  👤 玩家昵称    📊排行榜    🚪退出       │  ← 顶部用户栏
├──────────────────────────────────────────┤
│  🚩 010    ⏱ 00:45    😊                │  ← 游戏状态栏
├──────────────────────────────────────────┤
│                                          │
│         游戏棋盘区域                       │
│      (CSS Grid 布局)                     │
│                                          │
├──────────────────────────────────────────┤
│  初级 | 中级 | 高级 | 新游戏 | 我的记录   │  ← 底部控制栏
└──────────────────────────────────────────┘
```

### 8.2 颜色方案
- 背景：经典灰色 (#c0c0c0)
- 未打开格子：3D 凸起效果
- 已打开格子：凹陷效果
- 数字颜色：
  - 1: 蓝色, 2: 绿色, 3: 红色, 4: 深蓝, 5: 暗红, 6: 青色, 7: 黑色, 8: 灰色

### 8.3 页面结构
- **游戏主页面**：棋盘 + 状态栏 + 控制栏
- **排行榜弹窗/页面**：按难度切换的排行榜
- **我的记录弹窗/页面**：个人历史记录和统计
- **登录弹窗**：昵称输入框
